# Release Notes: v1.5.6-beta3

Release date: 2026-05-29

## Summary

`v1.5.6-beta3` adds safe administrator lifecycle management from the shared
Classic/Nexus `/admins` page.

## Highlights

- Classic and Nexus share the same administrator management page.
- Administrators can create another administrator with current-password
  confirmation, a new username, and a new password confirmation.
- Administrators can delete another administrator with current-password
  confirmation.
- Self-delete is blocked in the backend and reflected in the UI.
- Deleting an administrator removes that user's API tokens and reloads the
  APIV2 token cache immediately.
- Browser sessions for a deleted administrator become invalid because session
  validation now checks that the session user still exists in `users`.
- `/api/users` now includes `isCurrent` so the UI can hide destructive actions
  for the current administrator.
- `admin_created` and `admin_deleted` audit events are emitted with warn
  severity and without passwords, hashes, or raw token values.
- Nexus overview maps the new administrator audit events.

## Security Notes

- New duplicate administrator usernames are blocked at the service layer.
  No database unique index is introduced, because legacy databases may already
  contain duplicate usernames.
- Legacy duplicate self-delete checks are username-based: a target with the
  current session username cannot be deleted.
- Password checks for admin creation/deletion do not update `last_logins`.
- Deleted administrator API tokens stop working immediately after token cache
  reload.
- Deleted administrator cookie sessions are rejected on the next authenticated
  request.

## Upgrade Notes

- No manual migration step is required.
- Existing administrators, settings, clients, tokens, and sing-box config are
  preserved.
- After upgrade, use `/admins` in either Classic or Nexus UI to manage
  administrators.
